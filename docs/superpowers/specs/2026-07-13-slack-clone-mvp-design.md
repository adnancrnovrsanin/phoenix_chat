# Slack Clone MVP — Design Spec

**Date:** 2026-07-13
**Status:** Approved by user (brainstorming session)
**Codebase:** phoenix_chat (Phoenix 1.8, LiveView 1.2, Ecto/Postgres, Tailwind v4)

## 1. Overview

Turn the existing phoenix_chat app (currently an ephemeral WebRTC video-room tool) into a
Slack-style team chat MVP. The existing video-call machinery is retained as a minimal
"huddle" feature reachable from each channel. Visual design follows `DESIGN.md`
(IBM Carbon design system) strictly — including the all-light rule.

### Goals

- Persistent public channels and 1:1 direct messages with real-time delivery
- Users with accounts (phx.gen.auth) and unique usernames
- Emoji reactions (fixed palette) and per-channel unread badges
- Online presence indicators
- Huddle: one-click video room per channel (existing WebRTC code, restyled)
- UI in Serbian by default, English fallback, all strings through gettext
- Carbon look: IBM Plex Sans, 0px corners, hairline hierarchy, IBM Blue as scarce accent

### Non-goals (documented gaps, not accidents)

- Threads, message editing/deletion, file uploads, search, notifications
- Private channels, multi-workspace, channel admin roles
- Rate limiting (accepted MVP risk)
- Emoji picker library (fixed 8-emoji palette instead)
- Huddle presence visible from the channel view (huddle is a plain link)

## 2. Decisions log

| Question | Decision |
|---|---|
| Existing video-call code | Keep as "huddle" feature, minimal integration (button → room) |
| Auth | phx.gen.auth (LiveView flavor, as generated by Phoenix 1.8) + `username` |
| MVP extras | DMs, emoji reactions, unread badges. No edit/delete. |
| Architecture | Single `ChatLive` LiveView shell; `handle_params` channel switching; streams; PubSub topic per channel; `RoomChannel` reserved for huddle WebRTC |
| DM modeling | DM = row in `channels` with `kind: :dm` (unified messages/reactions/unread) |
| UI language | gettext, `sr` (Latin) default locale, English msgid fallback |
| Sidebar | Light (`#f4f4f4`) per DESIGN.md all-light rule — dark surface stays footer-only |
| Active nav item | Carbon selected-nav: `#e0e0e0` background + 2px IBM Blue left rule + 600 weight (no full blue fill; blue stays scarce) |

## 3. Data model

All timestamps `utc_datetime_usec`.

### `users` (phx.gen.auth + extension)

- Generated fields (email, hashed_password, confirmed_at, tokens table)
- `username` — string, required, unique (case-insensitive unique index on `lower(username)`),
  2–30 chars, regex `^[a-zA-Z0-9_.-]+$`. Chosen at registration; shown everywhere in chat.

### `channels`

- `name` — string, required. For `kind: :channel`: 2–40 chars, lowercased, regex
  `^[a-z0-9\-]+$`, unique among channels (partial unique index where kind = 'channel').
  For `kind: :dm`: set to the `dm_key` value (satisfies not-null; display name is derived
  from the other member at render time).
- `slug` — string, unique, URL identifier. For channels: same as `name`. For DMs: set to
  `"dm-" <> dm_key` (satisfies uniqueness; DMs are routed by the other user's username,
  never by slug).
- `topic` — string, optional, max 120 chars.
- `kind` — Ecto.Enum `[:channel, :dm]`.
- `dm_key` — string, nullable, unique index (partial, where kind = 'dm').
  Format `"<min_user_id>:<max_user_id>"` — guarantees one DM row per user pair.

### `channel_memberships`

- `channel_id`, `user_id` — FK, unique composite index.
- `last_read_at` — utc_datetime_usec, default now on insert.

### `messages`

- `channel_id`, `user_id` — FK, indexed (`channel_id, id` composite for pagination).
- `body` — text, required, 1–4000 chars after trim.
- No soft delete, no edited_at (MVP).

### `message_reactions`

- `message_id`, `user_id`, `emoji` — unique composite index (toggle semantics).
- `emoji` — string, validated against fixed palette:
  `👍 ❤️ 😂 🎉 👀 ✅ 🔥 🙏`

## 4. Contexts

### `Accounts` (generated + minor edits)

- phx.gen.auth as generated. Registration form/changeset extended with `username`.
- After successful registration: auto-join `#general` (hook in `register_user/1` flow).

### `Chat` (new)

Key public functions (names indicative):

- `list_joined_channels(user)` — memberships preloaded with unread counts, split by kind
- `get_channel_by_slug!(slug)`, `create_channel(user, attrs)` (creator auto-joins),
  `join_channel(user, channel)`, `list_browsable_channels(user)` (public not-yet-joined)
- `get_or_create_dm!(user_a, user_b)` — lazy DM creation via `dm_key` upsert
- `list_messages(channel, before_id \\ nil, limit \\ 50)` — newest page / older pages
- `send_message(user, channel, attrs)` — insert + broadcast `{:new_message, message}`
- `toggle_reaction(user, message, emoji)` — insert-or-delete + broadcast
  `{:reaction_changed, message_id, summary}`
- `mark_read(user, channel)` — bump `last_read_at`
- `unread_count(user, channel)` — messages newer than `last_read_at`, excluding own

Broadcasts go through `Chat.subscribe(channel)` / topic `"chat:channel:#{id}"` on
`PhoenixChat.PubSub`. Membership guard: message send / reaction / mark_read require an
existing membership (raise/error otherwise).

## 5. LiveView architecture

### Routes

```
live_session :require_authenticated_user (gen.auth on_mount):
  /            ChatLive :index      → redirects (push_navigate) to first channel (#general)
  /c/:slug     ChatLive :channel
  /dm/:username ChatLive :dm        → get_or_create_dm, renders like a channel
  /huddle/:slug RoomLive :show      → existing video room, room_id = channel slug

gen.auth routes as generated (/users/register, /users/log-in, /users/settings, ...)
```

`LobbyLive` (and its tests) are deleted — replaced by auth pages + ChatLive.

### `ChatLive` (single shell)

**mount:** load joined channels + DMs with unread counts; subscribe to
`chat:channel:#{id}` for every membership; track + subscribe `presence:online`
(key = user id, meta = username); assigns: sidebar lists, online MapSet, message form.

**handle_params:** resolve active conversation (by slug or username → DM);
`mark_read`; `stream(:messages, last_50, reset: true)`; update page title.

**Events:**

- `send_message` — sync insert via `Chat.send_message`; on success reset form
  (own message arrives via PubSub like everyone else's — single code path);
  on error render changeset error on the form
- `toggle_reaction` (message_id, emoji)
- `create_channel` (modal form) → on success navigate to new channel
- `join_channel` (browse modal) → join + navigate
- `load_older` — prepend older page with `stream_insert(..., at: 0)`

**handle_info:**

- `{:new_message, msg}` — active conversation → `stream_insert` + `mark_read`;
  inactive → increment that channel's unread badge in sidebar assigns, unless
  `msg.user_id` is the current user (own messages never count as unread — covers the
  two-tabs-on-different-channels case)
- `{:reaction_changed, message_id, summary}` — re-insert stream item with new summary
  (messages carried in stream entries include author + reaction summary)
- `presence_diff` — recompute online MapSet

### JS hooks (assets/js)

- `ScrollToBottom` — pin message list to bottom on new items (unless user scrolled up)
- `ComposerKeys` — Enter sends, Shift+Enter newline
- Existing `RoomRTC` hook unchanged (huddle)

Carbon tokens as CSS custom properties + Tailwind v4 `@theme`; `border-radius: 0`
everywhere (buttons, inputs, modals, avatars, chips); no drop shadows — hierarchy via
1px hairlines (`#e0e0e0`) and surface change (`#ffffff` ↔ `#f4f4f4`);
body letter-spacing `0.16px`.

**Layout:**

- **Sidebar 260px, `#f4f4f4`**, 1px hairline right edge. Workspace name 16/600 charcoal.
  Section labels ("Kanali", "Direktne poruke") sentence case, 12px, `#525252`.
  Items 14/400 `#161616`; hover `#e0e0e0`. Active item: `#e0e0e0` bg + 2px `#0f62fe`
  left rule + 600 weight. Unread badge: square tag, `#161616` bg, white 12/600 count.
  DM rows show a 8px square online dot (`#24a148` online / `#e0e0e0` hairline offline).
  "+" actions (new channel, browse) as ghost buttons, blue text, square.
- **Channel header 48px**, white, 1px hairline bottom. `#name` 14/600; topic 14/400
  `#525252` truncated. Right: "Pokreni huddle" as button-tertiary (white bg, 1px blue
  border, blue text, square, 12px/16px padding) — opens `/huddle/:slug` in new tab.
- **Message list**, white. Flat rows, hover `#f4f4f4`. Avatar 32px square, `#e0e0e0` bg,
  initials 14/600 `#525252`. Username 14/600, timestamp 12/400 `#8c8c8c`, body 14/400
  `#161616` with 0.16px tracking. Consecutive messages by same author within 5 min group
  (avatar/name only on first). Day dividers: 1px hairline with centered 12px date label.
- **Reactions:** square chips under message — 1px hairline border, emoji + 12px count,
  white bg; my active reaction: 1px `#0f62fe` border + blue count. Trailing "+" chip opens
  the 8-emoji palette (small square popover, 1px `#161616` border, no shadow).
- **Composer:** Carbon text-input — `#f4f4f4` bg, no side borders, 1px `#e0e0e0` bottom
  rule; focus replaces it with a 2px `#0f62fe` bottom rule (DESIGN.md signature focus
  treatment). Send = button-primary (solid `#0f62fe`, white, square). Sits on white with
  1px hairline top.
- **Modals** (create channel, browse channels): white, 1px `#161616` border, no shadow,
  title 20/400, inputs Carbon-style, primary/secondary buttons square.
- **Empty states:** 14/400 `#525252`, centered, no illustration.
- **Auth pages:** gen.auth templates restyled — display titles at weight 300
  ("Prijavi se", "Registruj se"), Carbon inputs, primary button full-width square.
- **Huddle room (`RoomLive`):** restyled — charcoal `#161616` full-bleed stage (app

**Gettext:** default locale `sr` (Latin script), all UI strings wrapped; English msgids
serve as fallback. Default locale set in config for the `PhoenixChatWeb.Gettext` backend
(exact config key per the gettext 1.x docs at implementation time).

## 7. Auth & huddle integration details

- phx.gen.auth run as generated (magic link + optional password, LiveView pages, scope
  pattern). `username` added: migration, changeset validation, registration form field,
  uniqueness error message.
- Registration success → `Chat.join_channel(user, general)` (idempotent).
- `RoomLive` mount moves under the authenticated live_session; `display_name` becomes
  `current_scope.user.username`; the `?dn=` query param and its empty-name redirect are
  removed. Room capacity 6 stays.
- Huddle button links to `/huddle/:slug` (`target="_blank"`), room id = channel slug, so
  channel-mates land in the same room. No huddle state shown in ChatLive (non-goal).
- **The signaling transport is membership-gated end to end:** `RoomLive` mints a
  `Phoenix.Token` (`"user socket"` salt, 1-day max age) per render and hands it to the
  `RoomRTC` hook via `data-token`; `UserSocket.connect/3` verifies it and assigns
  `user_id`; `RoomChannel.join/3` loads the user and channel and rejects non-members
  with `error:unauthorized`. `display_name` is derived server-side from the username —
  client-supplied values are ignored (no identity spoofing over the socket).

## 8. Error handling

- Changeset validation on message body, channel name, topic, username.
- DB unique constraints (username, channel name, dm_key, membership, reaction) surfaced
  as friendly gettext'd changeset errors — never 500s.
- Message send is synchronous in `handle_event` — failures render on the composer form.
- Cross-membership access (URL guessing to a channel not joined): channel view for
  public channels is join-gated — visiting a not-joined public channel shows a
  "Pridruži se" interstitial instead of messages; DMs 404 unless participant.
- LV reconnect self-heals: fresh mount reloads sidebar + messages from DB.
- Known accepted risks: no rate limiting; PubSub at-most-once delivery (a dropped
  broadcast is corrected on next mount/navigation).
- ~~WebRTC signaling transport not membership-gated at the socket layer~~ **Resolved:**
  `UserSocket.connect/3` now verifies a signed `Phoenix.Token` and `RoomChannel.join/3`
  checks `Chat.member?/2`, rejecting unauthenticated or non-member connections with
  `error:unauthorized` (see §7).

## 9. Testing

- **Context tests (`Chat`):** channel creation + slug validation, join idempotency,
  DM dedup (both orderings return same row), unread math (own messages excluded,
  mark_read resets), reaction toggle (add/remove/unique), membership guards.
- **LiveView tests (`ChatLive`):** message send renders for a second connected LV in the
  same channel; unread badge increments for member viewing another channel; switching
  channels marks read; reaction chip toggles; create-channel modal navigates; join gate
  for not-joined channel; DM creation via `/dm/:username`.
- **RoomLive tests:** updated for auth-based mount (existing tests reworked, `?dn=` cases
  removed).
- **Auth tests:** as generated + username validation cases.
- Gate: existing `mix precommit` alias (compile --warnings-as-errors, format, test).
